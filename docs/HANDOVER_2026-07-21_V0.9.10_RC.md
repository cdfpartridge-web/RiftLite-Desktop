## What Was Audited And Fixed

### Accounts and cloud sync

- Account identity changes are generation-pinned so an in-flight request cannot write under a newly linked account.
- Cloud restore is fenced against local edits, account changes, queue races, remote disappearance, and unapproved divergence.
- Failed restores preserve the pending local-sync reason without resurrecting stale work over newer mutations.
- Secure credential storage, reconnect, replacement, and migrated identity-alias paths fail closed.
- Website account gates accept only durable or server-proven canonical aliases; unrelated and anonymous identities cannot inherit hub, replay, or Discord authority.
- Discord verification-code redemption is transactional, one-time, retry-safe for the same user, and rejects a cross-account race.
- Discord OAuth recovery reads all relevant guild links instead of relying on an ambiguous 50-record slice.

### Web Replay

- Atlas and TCGA capture remain isolated provider pipelines but converge on the authenticated Replay V2 library/player.
- TCGA finalization can attach directly to the active match even when no local `ReplayRecord` parent exists.
- Match drafts persist the exact Web Replay ID, owning account UID, and local replay association across restart/retry.
- Replay upload/finalization/deletion is account-pinned and guarded against deleted or replaced parent matches.
- Migrated accounts can resume deterministic replay IDs only through server-proven aliases; the server never probes unproven UIDs.
- Raw artifacts remain private. Only processed canonical replay output follows Private, Unlisted, or Public visibility.
- Closing the final match review releases capture/provider switching; failed review deletion stays visible and retryable.
- Public replay cards use processed canonical matchup identity when an older generated title is stale.
- The public replay archive uses stable cursor pagination instead of ending after the first 48 records. Additional batches append without duplicates, preserve loaded cards on failure, and can be retried safely.
- TCGA points, counters, grouped/equipped attachment movement, battlefield ownership, and replay presentation are covered by regression tests.
- TCGA battlefield resolution accepts validated alternate artwork hashes. The Ambessa capture's otherwise unnamed hash now resolves to `VEN-163` / **Risen Altar** in both the replay and match-reporting paths.
- TCGA opening-hand playback prefers the capture player's first real hand instead of stopping at an earlier opponent-only hidden-hand state. Exact perspective mulligans are reconstructed only when unique IDs, ordered deck boundaries, set deltas, and redraw counts prove the outgoing and incoming cards; ambiguous captures retain a truthful count-only fallback.
- TCGA BO1 scores now mirror Atlas: the confirmed match-logger result is authoritative for the series record (`1-0`, `0-1`, or `0-0`), while captured Riftbound points remain on the individual game result and board. Discord evaluates the authoritative BO1 winner/outcome before legacy point-valued series fields, so older canonicals also report correctly without changing BO3 behavior.
- The affected Ambessa replay `rl2_5fd0b45bc42bb42587e4ecb8dccda930` was atomically reprocessed from its checksum-verified private raw artifact after deployment. The ready/unlisted record now serves canonical SHA-256 `5240F6966E66397EE9C9AE8323105AECAE704B490D3BB4A5E76D3ECDF84ED819`, with 496 events, 21 checkpoints, the exact two-card mulligan at event 46, and zero integrity/privacy issues. The previous immutable artifact was retained for rollback, and no historical Discord post was generated.
- TCGA Discord sharing carries an integrity-bound snapshot of the selected hubs from capture start, intersects it with completion/current membership, and reloads consent immediately before every post or retry. Future eligible captures auto-post through the existing hub configuration; pre-fix captures without start-time evidence require an explicit manual share.
- Two-perspective combining and its consent model remain Atlas-only.

### Match reporting and aggregation

- Battlefield ownership uses seat evidence consistently between Web Replay and the match logger.
- BO3 combination preserves exact battlefield IDs/images and does not inherit one child game's Web Replay identity.
- Undo-combine is remote-aware, account-pinned, concurrency-safe, retryable, and updates public, hub, team, and aggregate state without orphaning originals.
- Public aggregate append ignores forged client match bodies and uses the authoritative stored match under canonical identity.
- Root public matches are included in identity migration.
- Private-hub aggregate upsert/delete validates action/path, stored ownership, canonical identity, and server-owned deletion indexes. The source-delete-first desktop flow remains supported without allowing one member to manipulate another member's counters.
- TCGA confirmation is local-first: the local match, product capture, replay index, and replay association commit before the popup returns. Upload, processing, Discord delivery, and Firebase match/hub sync continue in the background through a durable startup/two-minute retry manifest; Atlas retains its existing synchronous confirmation behavior.

### Desktop lifecycle and packaging

- Updater install handoff is guarded so a downloaded installer can actually own the quit/restart path.
- Atlas shell recovery, game-chat focus recovery, macOS activate/window lifecycle, and display-media callback safety were reviewed and regression-tested.
- Windows packaging is now fail-closed: identity/version, embedded update feed, installer metadata, NSIS archive, FFmpeg execution/license, manifest size/SHA-512, ASAR identity, and the full regenerated differential blockmap must all match.
- Packaged smoke mode requires an explicit absolute temporary root and redirects user data, documents, pictures, downloads, temp, and crash dumps beneath it.
- Smoke mode blocks non-local renderer networking and disables updater, analytics, replay uploads/backfills, simulator listener, overlay listener, and global hotkeys.
- Smoke success requires a real renderer shell, non-zero layout, Home navigation, preload bridge, meaningful DOM text, screenshot, clean startup log, and exit code zero.
- macOS packaging stages and architecture-checks separate x64/arm64 FFmpeg binaries in the actual app bundle.
- The central Mac packager overrides the Windows GitHub feed with a tested object-form Mac feed. A live electron-builder effective-config test protects this subtle merge rule.
- Mac verification checks four DMG/ZIP entries and hashes, executable/FFmpeg architecture, bundle identity/version/protocol, embedded updater feed, strict code signature, licenses, and DMG integrity before upload.

- A local website server without the production Vercel Blob credential can list public replay metadata but cannot fetch production canonical artifacts. Validate artifact playback on the deployed environment or with an authorized local Blob configuration; do not copy production credentials into source.
- Discord replay posting is configured and the bot/guild/commands now agree. Test hubs must configure `reports_channel`; automatic replay posts intentionally do not fall back to `feed_channel`.

Add the secret and register `https://www.riftlite.com/api/auth/discord/callback` in the Discord Developer Portal before testing **Continue with Discord**.
