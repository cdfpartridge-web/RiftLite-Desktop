# RiftLite Beta 0.7.98 (Local Installer)

## Discord replay sharing

- Selecting a private hub is now the complete replay-sharing opt-in; there is no separate master switch to miss.
- Future shared Atlas replays upload as Unlisted and post to the selected hub's configured Discord `reports_channel`.
- Hub IDs are displayed beside hub names so similarly named or older hub entries can be distinguished.
- Existing uploaded replays have an explicit **Share to Discord** / retry action. RiftLite confirms before changing a Private replay to Unlisted.
- Account ownership, current hub membership, configured-channel checks, and duplicate-delivery protection remain enforced by the website.

This installer is local-only. It is not a GitHub release and does not change the public updater release.
